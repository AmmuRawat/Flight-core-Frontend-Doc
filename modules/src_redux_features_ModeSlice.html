Module src/redux/features/ModeSlice

Description

This slice manages the state related to the drone’s operation and mission planning. It includes:

  • info: Mission metadata (e.g., mission name, creation timestamp).
  • actionMode: The current action mode (derived from actionLabels).
  • displayMode: The display mode for the UI.
  • homeCoords & gcsCoords: Coordinates for the drone’s home and Ground Control Station.
  • guideWaypoint & tempGuideWaypoint: Waypoints used for guiding the drone.
  • missionWaypoints: An array of mission waypoints, each with an ID.
  • deletedMissionWaypoints: A list of removed waypoints (for undo functionality).
  • travelledMissionWaypoints: An array tracking the drone’s travelled path.
  • roiWaypoint: A region-of-interest waypoint.
  • singleMode: The operating mode in single mode ("Guided", "Track", or "ROI"). It provides a comprehensive set of reducers to:
  • Update mission information.
  • Manage waypoints (add, update, remove, reset, and undo removals).
  • Set coordinates for home and GCS.
  • Control single mode and ROI waypoint. Selectors are exported to access various parts of the state.

Example

// Adding a new mission waypoint:
dispatch(addMissionWaypoint({
  coords: [lng, lat],
  altitude: 50,
  speed: 5,
  loiterTime: 0,
  droneAction: { onReachingAction: 'RTL' },
  cameraAction: { triggerTime: 10, triggerTimeAction: 'Take Photo', onReachingAction: 'Start Video' }
}));
// Accessing mission information:
const missionInfo = useSelector(selectMissionInfo);
