Module src/redux/store

Description

This module sets up the Redux store using Redux Toolkit's configureStore and integrates state persistence via redux-persist. The root reducer combines multiple slice reducers responsible for different parts of the application's state:

  • mode: Manages drone mode and related settings.
  • map: Manages map-related state.
  • setting: Contains nested reducers for default settings, camera settings, and failsafe settings.
  • savedMission: Manages saved mission data.
  • offlineMap: Handles offline map state.
  • app: Global application state.
  • geoFence: Geofence configuration.
  • trigger: Trigger events for various actions.
  • screen: Screen-related state.
  • actionButton: Configuration for action button behavior. The persist configuration uses AsyncStorage and whitelists only the setting and actionButton slices. Middleware is customized to disable serializable and immutable state checks, and thunk middleware is added. After persisting the store, a callback dispatches setLoadedState(true) after a 5-second delay, ensuring the application renders only after the persisted state is properly restored.

Example

// In your root component:
import { Provider } from 'react-redux';
import { PersistGate } from 'redux-persist/integration/react';
import { store, persistor } from './store';
function App() {
  return (
    <Provider store={store}>
      <PersistGate loading={null} persistor={persistor}>
        <YourRootComponent />
      </PersistGate>
    </Provider>
  );
}

Returns

An object containing the configured Redux store and persistor.
